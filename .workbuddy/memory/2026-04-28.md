## 首页计数器 0 问题审查与修复

根因：`getCachedPublicWordsCountValue()` 使用 `head:true` + `select("*")` 导致 PostgREST HEAD 请求不返回 Content-Range，`count=null` → `count??0` → 返回 0。
Featured Words 使用普通 GET `select(WORD_SELECT)`，不受影响，因此首页能显示 6 条但计数器为 0。

修复（commit 58e1869）：
1. `select("*", { count:"exact", head:true })` → `select("id", { count:"exact" }).limit(1)`
2. 缓存键 `public-word-count` → `public-word-count-v2`
3. 缓存键 `public-landing-snapshot` → `public-landing-snapshot-v2`

验证：
- 110/110 tests passed ✅
- ESLint clean ✅
- Build passed，首页仍为 Static + ISR 5m ✅
- 线上 RSC payload 包含 `"target":1793` ✅（web_fetch 显示 0 是因为 AnimatedCounter 初始 useState(0)，客户端 JS 加载后动画到 1793）
- 生成审查报告：homepage-counter-zero-audit.md

## AnimatedCounter SSR 语义修复

根因：AnimatedCounter 用 `useState(0)` 初始化，SSR HTML 渲染为 `<span>0</span>`，导致 web_fetch/curl/搜索引擎均看到 0。
修复（commit 1740221）：
1. `useState(0)` → `useState(target)` — SSR 直接显示真实值
2. `isFirstRender` ref + guard — 跳过 mount 动画，避免 hydration 后闪烁
3. `target` 变化时仍保留动画能力
